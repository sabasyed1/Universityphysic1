Motion in two dimensions - Concepts of vectors¶

The Objective:¶

The objective of this lab is to study motion in two dimensions and find out how velocity vector can be broken into components to study the motion. To illustrate this we will be using the motion of a projectile.¶

The Introduction:¶

In this experiment we will launch a projectile (a metallic ball) by using an inclined plane of the side of a table. The experimental setup is shown below. Due to the inclination angle the ball will accelerate down the plane. Once it reaches the edge of the table it will drop off freely under the force of gravity.

However, the ball will not fall vertically down as we observed in the case of the dropping of the metallic ball in the first experiment. Instead it will follow a parabollic trajectory as shown in the diagram below. This is due to the fact that the ball actually has a velocity at the time it is leaving the surface (unlike in the first experiment where it was merely dropped from a 2m height). This velocity $(v_f)$ is along the plane. Since velocity is a vector quantity, It can be decomposed into two components, a horizontal component ,$v_f\cos\theta$, and a vertical component, $v_f\sin\theta$, as shown in the figure. The horizontal component of the velocity will continue to take the ball away from the table while the ball is falling under gravity. That is the reason it does not fall vertically down.

We will see how the component of the velocity will result in change in the result of how far the ball will be launched away from the table.

Theory:¶

Horizontally launched projectiles:¶

When a projectile is launched from a height horizontally, its component of velocity in the vertical direction is zero, and the entire velocity is in the horizontal direction at the initial point. Earth's gravity will give the projectile a velocity in the vertical direction, however that will have no effect on its velocity in the horizontal direction. Thus, the projectile will continue to have the same horizontal velocity its entire time of flight, and that will determine how fast it will be moving in the horizontal direction. However, the vertical motion will determine how long this motion will last, since once the projectile hits the ground, the motion will cease. Thus, the duration of the flight is given by the time the projectile takes to hit the ground. This is given by $t = \sqrt{\frac{2H}{g}}$, where $H$ is the height from there the projectile is launched. Therefore we the distance the projectile will travel from the point it is launched is given by $D = v_ft = v_f\sqrt{\frac{2H}{g}}$.

Projectile launched from an incline plane:¶

However, our case is slightly more complicated. The projectile is launched from an inclined plane, and therefore not horizontally. As a result of that the time of flight needs to be modified using the kinematic equation, $H = v_fT\sin\theta + \frac{1}{2}gT^2$. One can calculate the value of the time of the flight, $T$, by solving the above quadratic equation, giving us,

$T = \sqrt{\frac{v_f^2\sin^2\theta}{g^2} + \frac{2H}{g}} - \frac{v_f\sin\theta}{g}$.¶

Hence the corrected range of the projectile (taking into account the angle of inclination) is:

$D = v_f\left[\sqrt{\frac{v_f^2\sin^2\theta}{g^2} + \frac{2H}{g}} - \frac{v_f\sin\theta}{g}\right]\cos\theta$.¶

def compute_final_velocity(roll_times, length=L):
    '''
    This function takes as input the slide times and spits out the value of the
    final velocity.
    
    Arguments: 
    roll_times :: This should be a numpy array
    length :: The length of the slide. By default it will pick up the length you provided above.
    
    Example function call:
    x = np.array([2.87, 2.9, 2.93, 2.91, 2.88, 2.87, 2.94, 2.9, 2.91, 2.84])
    compute_final_velocity(x)
    >> 0.8290155440414507
    '''
    average_slide_time = np.mean(roll_times)
    v_avg = length/average_slide_time
    vf = 2*v_avg

    return vf



def compute_range(height, angle, final_velocity, g=g):
    '''
    This function takes as input the height of the launching point above ground (H), and 
    spits out the value of value of the range.
    
    Arguments:
    height :: The height (H) in meters of the point of launching above the ground.
    angle :: Angle of the incline in degrees that was used to roll the ball
    final_velocity :: Final velocity computed from the previous function call
    g :: Acceleration due to gravity. Default value will be pick from cell above
    '''
    
    time_of_fall = (2*height/g)**0.5
    angle = np.deg2rad(angle) # Convert the angle to radians
    R = final_velocity*time_of_fall
    
    R_corr = (1/g)*((final_velocity*np.sin(angle))**2 + 2*g*H)**0.5 - final_velocity*np.sin(angle)/g
    R_corr = R_corr*final_velocity*np.cos(angle)
    
    return [R, R_corr]

Fractional error¶

Frctional error is calculated by taking the difference of the experimental value from the true value and then dividing it by the true value. In this experiment the true value is not known, so we are going to calculate the fractional error w.r.t between the measured value and the calculated value. We are going to pretend that the measured value D_measured is the true value, and hence the resulting fractional error is:

$\Delta R = \frac{D_{\rm calculated} - D_{\rm measured}}{D_{\rm measured}}$.¶

We are going to calculate this quantity for both the approximate result and the corrected result.The expectation is that the difference between the two should be small, but the fraction error should be lower for the corrected case. The reason we expect the results to be very close is because the value of the angle very small in general. Thus the ball is being lanunched almost horizontal. The approximate result assumes that the ball is launched horizontally, which is not very wrong under the small-angle assumption.

Uncertainty Analysis¶

We will compute the fractional uncertainty in our experiment using error propagation technique. Let us apply that for the simpler case of the $D_{\rm approx}$. The expression we need to use is $D = v_f\sqrt{\frac{2H}{g}}$. We take natural logarithm on both sides to get, $\ln D = \ln v_f + \frac{1}{2}\ln \frac{2H}{g} = \ln v_f + \frac{1}{2}(\ln 2 + \ln H - \ln g)$. How of course, since $2$, and $g$ are constants, we can take a derivtive on both sides, to get $\frac{dD}{D} = \frac{dv_f}{v_f} + \frac{1}{2}\frac{dH}{H}$. From this we can write the fractional uncertainty in the measurement of the distance $D_{\rm approx}$ is

$\frac{\Delta D_{\rm approx}}{D_{\rm approx}} = \left|\frac{\Delta v_f}{v_f}\right| + \frac{1}{2}\left|\frac{\Delta H}{H}\right|$,¶

where, $\Delta H$ is the least count of the meter scale that we used to measure the distance to the spot on the ground where the projectile hit from the point vertically beneath the launching point. This is is typically $= 1~{\rm mm}$. To find $\left|\frac{\Delta v_f}{v_f}\right|$, we note that $v_f = 2v_{\rm avg} = \frac{2L}{t}$, where $L$ is the full length the projectile rolled on the inclined plane, and $t$ is the time it took to roll. Using the same argument above, $\left|\frac{\Delta v_f}{v_f}\right| = \left|\frac{\Delta t}{t}\right|$. Thus, the fractional error in the measurement of the approximate distance $D_{\rm approx}$ is,

$\frac{\Delta D_{\rm approx}}{D_{\rm approx}} = \left|\frac{\Delta t}{t}\right| + \frac{1}{2}\left|\frac{\Delta H}{H}\right|$,¶

where $\Delta t$ is the least count in the stopwatch, which in the case of the lab-supplied stopwatch is $10~{\rm ms}$.

We can of course do the same analysis for the corrected result, but the expression will be too complicated, and is unnecessary, since that will just add the error in our angle measurement on top of this, which is not the main source of error.
